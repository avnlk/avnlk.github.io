[context] what is the triangular interleaver?

In 5G-NR communications, interleaving protects data against burst errors by shuffling bits before transmission. The 3GPP specification dictates writing data row-wise into an isosceles triangle and reading it column-wise.

[architecture_evolution]

The path to 0.41µs required completely dismantling the 3GPP algorithm to align with FPGA hardware primitives.

Phase 1: The 3GPP Spec (Math Bottleneck)

The Naive Approach: Algorithm 1 in TS38.212 requires determining a side length $T$ such that T(T+1)/2 ≥ E, then executing nested for loops that dynamically track and skip (T(T+1)/2) - E null bits.

Why we didn't build it: Solving quadratic inequalities and running deep nested loops with dynamic variables is hostile to hardware. Synthesis would generate massive ALUs (DSP slices) and multi-microsecond computation stalls. This approach was rejected prior to synthesis.

Phase 2: The Algorithmic Baseline (FSM Bottleneck)

The Optimization: We designed a baseline that precomputed row-offsets into a BRAM array (row_start) to avoid dynamic division and quadratic math at runtime. The data path used a state machine to track coordinates (r, c) and skip nulls via simple addition: k_src = row_start[r] + c.

The Limit: While mathematically optimized, the state machine created a sequential data dependency. The hardware could only resolve the destination for 1 bit per clock cycle. Processing 512 bits required over 500 cycles.

Phase 3: The Final Architecture (O(1) BRAM Parallelism)

The Breakthrough: The structure of the triangle for any fixed block size $E$ is completely deterministic. We replaced the state machine entirely by calculating all valid row-to-column permutations at compile-time and storing them in a 2D ROM array (PERM_TABLE).

The Result: By breaking the sequential dependency, we utilized #pragma HLS UNROLL to query the BRAM 16 times concurrently. The FPGA now asks the memory for the routing map and moves 16 bits per clock cycle, achieving true wire-speed saturation of the AXI4-Stream bus.

[engineering_deep_dives]
Trade-offs: Silicon Area vs. Execution Speed

The Phase 3 architecture intentionally trades BRAM footprint for execution speed. By hardcoding the indices into a PERM_TABLE, we consume 3 extra BRAM-18K tiles on the Spartan-7 die compared to the baseline. However, in 5G baseband processing, on-die memory is cheap compared to the critical value of nanoseconds saved in the radio path. We also free up DSP slices that would have been wasted on math operations.

Verification Methodology
3GPP Corner-Case Validation
Design correctness was verified against the 16 golden test vectors provided in the 3GPP assignment specifications. The Verilog RTL output was bit-matched through C/RTL Co-simulation, covering all payload sizes from E=16 to E=512 with zero margin for error across all triangular null-skipping conditions.
